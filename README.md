# market-book-toc-title-writer

为中文市场书、商业书、大众非虚构和实用类图书设计、改写、评审节标题与小节标题的 Skill。核心能力是把平淡、教材式或说明书式目录，重构为具有市场张力与点击动机的标题，同时守住原内容语义、全书逻辑和可兑现性；也支持从章级框架继续展开节或小节，以及诊断标题同质化与内容重复。

## 它能做什么

- **生成**：依据全书结构、正文内容与篇幅约束，产出节（`1.1`）或节＋小节（`1.1.1`）标题。
- **改写**：对用户现成标题执行"语义继承、句法清零"——继承原意与专有名词，不继承原句式的分句顺序、句法骨架和修辞方式。
- **评审**：从纵向层级、横向协调、内容可兑现、去重、市场表达、篇幅六个维度检查目录质量。
- **诊断**：识别标题同质化、概念重复占位、编号与层级异常。

## 设计原则

1. 市场张力来自读者欲望、现实冲突、结果变化和身份代入，不等于口语化，也不得用刺激性措辞掩盖内容空缺。
2. 改写时继承原意，不继承原句式；只处理节与小节，除非用户明确要求调整更高层级。
3. 不得添加框架和材料中没有依据的数据、案例、机构、结论或效果；缺少来源时标注"待核查"，不得补造。
4. 不承诺"爆款"。"爆款感"只表示标题使用了经过验证的点击动机和信息结构。

## 目录结构

```text
market-book-toc-title-writer/
├── SKILL.md                      # 技能主文件：任务边界、改写模式、生成与评审流程
├── agents/
│   └── openai.yaml              # 代理配置
├── references/
│   ├── intake-and-planning.md   # 必要输入清单与规划方法
│   ├── structure-and-length.md  # 篇幅与层级规划规则
│   ├── title-mechanics.md       # 标题机制与钩子取舍顺序
│   ├── pattern-library.md       # 标题模式库
│   └── quality-review.md        # 评审六维检查清单
└── scripts/
    └── audit_toc.py             # 目录审计脚本（编号、重复、长度、刺激词、层级异常）
```

## 使用方法

1. 在对话中调用本 Skill，提供图书选题方向、目标读者、已确定的章级框架、输出深度（节／小节）及篇幅约束。
2. Skill 会按需读取 `references/` 下的规则文件，并在生成前为每个内容单元建立内容契约。
3. 若已将目录保存为 Markdown 或纯文本，可运行 `scripts/audit_toc.py` 做形式风险检查：

```bash
python scripts/audit_toc.py <你的目录文件>.md
```

> 脚本只能发现形式风险，不能替代内容判断。
