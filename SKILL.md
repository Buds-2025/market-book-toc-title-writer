# 市场书目录标题写作

从全书结构和正文内容中生成节与小节标题。先提炼每个内容单元的语义契约，再用爆款标题机制重构表达。市场张力来自读者欲望、现实冲突、结果变化和身份代入，不等于口语化，也不得用刺激性措辞掩盖内容空缺。

## 明确任务边界

- 将“篇／部分／阶段”和“章”视为已知框架，默认不重写这些层级。
- 只生成到用户指定的深度：选择“节”时停在`1.1`；选择“小节”时生成`1.1`和`1.1.1`。
- 用户要求评审或改写现有目录时，保留已经确认的知识范围、专有名词和编号体系。
- 发现章级框架存在断层、重复或顺序冲突时，先指出具体问题并给出最小调整建议，不擅自重构全书。

## 改写模式先做语义脱壳

用户提供现成节标题并要求改写时，执行“语义继承、句法清零”：

1. 将原题还原为一张语义卡，只保留对象、读者问题、方法或动作、目标结果、适用场景和必要专有名词。
2. 标记必须保留的事实词，例如产品名、工具名、交付物、专业概念和确定数字；其余措辞均不视为约束。
3. 暂时隐藏原题，依据语义卡重新命名。不得沿用原题的分句顺序、句法骨架、开头结构或修辞方式。
4. 生成后再与原题核对语义，确认没有增删主要内容，也没有把局部功能扩大为整体效果。
5. 若新题只是在原句中替换近义词、增加口语词或调整标点，判定为无效改写，重新生成。

原题只提供“写什么”，不提供“怎么写”。专有名词可以复用，行文风格不得默认继承。

## 先补齐必要输入

读取[references/intake-and-planning.md](references/intake-and-planning.md)。检查对话和附件中已经存在的信息，只追问会实质改变目录的缺项。首次追问集中在一轮内完成。

至少确认：

1. 图书选题方向、暂定书名或一句话内容承诺；
2. 目标读者、读者当前状态和读完后的目标状态；
3. 已确定的篇／部分／阶段与章级框架，以及必须覆盖的内容；
4. 输出到“节”还是“小节”；
5. 目标正文总字数、章节数量或出版社篇幅限制。无法提供时，改用相对权重规划，不虚构精确字数。

语气、竞品差异、案例来源、禁用词和作者人设会明显影响结果时一并确认。用户已经给出答案时不得重复提问。

## 建立全书结构模型

1. 用一句话概括全书承诺，确认目录服务于同一读者转变。
2. 为每章写出“进入状态、章任务、离开状态”，检查前后承接。
3. 将每章拆成若干内容任务。每个任务只解决一个主要问题，并能支撑一个独立的节标题。
4. 标记概念、方法、步骤、案例、工具、练习和复盘，避免把不同颗粒度内容并列。
5. 检查同一概念是否在多章重复出现。必须重复时，明确本次承担的是定义、应用、进阶还是复盘。

## 规划篇幅与层级

读取[references/structure-and-length.md](references/structure-and-length.md)，先按章节任务分配相对权重，再换算字数和内容单元数量。

- 依据读者价值、理解难度、行动复杂度和证据负荷分配篇幅，避免机械平均。
- 节层默认承载一个完整问题、方法或阶段结果；小节层承载一个可单独讲清的概念、动作、判断或案例。
- 只有两个并列项时，先判断能否合并；超过六个并列项时，先判断是否需要聚类或增加中间层。
- 用户只要求节标题时，在内部规划内容边界即可，不输出未经要求的小节。
- 篇幅不足以兑现标题承诺时，缩小承诺或合并内容；不得只保留有吸引力的标题。

## 为每个单元建立内容契约

生成标题前，为每个节或小节确定：

- 它回答的读者问题；
- 它给出的主要结论、方法或结果；
- 它使用的事实、案例、工具或步骤；
- 它与父级标题及相邻标题的关系；
- 它明确不覆盖的内容；
- 建议篇幅或相对权重。

内容契约用于约束标题。缺少正文事实时，只写可由现有框架支持的承诺，并将待补证据标记为待确认。

## 生成节与小节标题

始终读取[references/title-mechanics.md](references/title-mechanics.md)和[references/pattern-library.md](references/pattern-library.md)。在内部为每个内容单元至少生成六个实质不同的候选方向，再选择最适合全书结构的一项。六个方向至少覆盖：结果前置、痛点冲突、低门槛获得、身份或能力跃迁、损失规避、场景或品牌锚点。不能只替换近义词。

每个候选只使用一个主钩子。优先组合“高辨识对象＋变化或冲突＋读者行动＋可感知结果”，再根据层级压缩。爆款公式用于组织信息和点击动机，不用于堆叠热词。

按以下顺序取舍，后项不得破坏前项：

1. 内容语义与范围可兑现；
2. 层级任务和全书逻辑；
3. 读者能识别的欲望、痛点或具体收益；
4. 冲突、反差、变化、代价或身份代入；
5. 对象、场景、动作或结果的明确程度；
6. 全书语气及同级标题一致性；
7. 标点与修辞。

节标题必须形成明确的点击理由，可以承担较强的读者问题、冲突或阶段结果；小节标题应更具体、更可执行，并共同覆盖父级标题的承诺。不得把“通俗易懂”误当成市场张力，不得把所有标题写成同一种句模，也不得为了变化破坏同级的语法关系和颗粒度。

## 执行硬性规则

- 一个标题只保留一个主要承诺。补充分句只说明范围、证据、方法或结果。
- 不得添加框架和材料中没有依据的数据、案例、机构、结论、效果或作者经历。
- 默认避开“重磅”“炸裂”“杀疯了”“震惊”“颠覆”“必看”“终极”“秘籍”等空泛刺激词。
- “唯一”“第一”“最强”“彻底解决”等绝对表述必须有可靠证据，并提醒用户核查来源。
- 疑问句必须由对应单元给出明确答案。同组标题不连续堆叠疑问句。
- 第一人称只用于作者真实经历构成证据，且全书采用相应叙述口吻的情况。
- 时事、品牌和流行词只有在内容本身依赖这些对象时才能进入标题，不把短期热度当作目录价值。
- 标题与正文的事实、数据和案例需要核查来源。缺少来源时标注“待核查”，不得补造。
- 默认使用中文全角标点，避免连续感叹号、省略号、斜杠堆叠和符号装饰。
- 不得照着原题做口语化翻译。出现“快速、轻松、更清楚、更高效”等泛化收益时，必须补足具体对象、冲突、变化或结果。
- 不承诺“爆款”。“爆款感”只表示标题使用了经过验证的点击动机和信息结构。

## 评审并重写

最终输出前读取[references/quality-review.md](references/quality-review.md)，依次执行：

1. 纵向检查：篇／章／节／小节是否形成包含关系和推进关系；
2. 横向检查：同级标题的颗粒度、语法、语气和吸引力是否协调；
3. 内容检查：每个标题是否有正文内容可兑现，父级承诺是否被子级覆盖；
4. 去重检查：概念、方法、案例和结果是否重复占位；
5. 市场表达检查：是否同时具备点击理由、读者代入、具体收益和可信边界；
6. 篇幅检查：建议字数是否支撑任务，标题数量是否与全书容量相称。

若目录已保存为Markdown或纯文本，可运行`scripts/audit_toc.py`检查编号、重复、标题长度、刺激词和部分层级异常。脚本只能发现形式风险，不能替代内容判断。

## 默认输出格式

根据任务规模控制说明长度，默认提供：

1. `输入摘要与假设`：列出已确认信息和会影响结果的暂定假设；
2. `篇幅与结构规划`：按章给出任务、相对权重、建议字数和节数；
3. `目录正文`：使用用户指定的编号，输出节标题或节＋小节标题；
4. `内容边界`：用一句话说明各节写什么，必要时标注建议字数；
5. `逻辑与风险提示`：指出顺序、重复、证据或篇幅风险，并提醒核查事实来源。

用户只要目录标题时，只输出改写后的目录和必要的事实核查提示，不附带候选生成过程。用户要求多个方案时，优先提供“稳健市场版”和“高张力版”，两版都必须具备市场吸引力，差别只在钩子强度。
