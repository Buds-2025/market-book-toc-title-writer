## 必要输入

创建目录前至少获得以下信息：

| 信息 | 用途 | 缺失时的处理 |
|---|---|---|
| 选题方向或一句话承诺 | 判断全书边界 | 必须追问 |
| 目标读者及其起点 | 判断术语、难度和关注点 | 必须追问 |
| 章级框架 | 确定节与小节的位置 | 必须追问 |
| 输出深度 | 决定停在节或继续到小节 | 必须追问 |
| 目标正文总字数或篇幅限制 | 分配章节和内容单元 | 可先用相对权重 |
| 作者口吻及市场定位 | 控制标题刺激度与专业度 | 可提出暂定假设 |
| 必须覆盖或禁止覆盖的内容 | 防止范围偏移 | 有迹象时追问 |
| 已有材料、数据和案例 | 判断标题承诺能否兑现 | 缺失时标记待补 |

章级框架至少应包含章名和每章准备解决的问题。只有章名且含义宽泛时，要求用户为每章补充一句内容说明。

## 一轮追问模板

只询问对话中尚未出现的项目，并集中为一轮：

1. 请用一句话说明这本书帮助哪类读者解决什么问题，暂定书名是什么？
2. 请提供已经确定的篇／部分／阶段与章级框架，并为每章补充一句内容范围。
3. 本次目录生成到“节”还是“小节”？
4. 计划正文总字数、页数或出版社给定篇幅是多少？如果尚未确定，是否接受先按相对权重规划？
5. 目录希望偏稳健专业、通俗有张力，还是两者平衡？有哪些必须使用或避免的表达？
6. 有哪些案例、数据、方法或作者经历必须进入目录？它们的来源是否已经确认？

不得要求用户重复已经提供的信息。用户只提供部分答案时，先用可逆假设继续，只有缺项会改变全书结构时才再次追问。

## 三种工作模式

### 从章级框架新建

先建立章节任务图，再规划篇幅、节和小节。默认输出一套完整目录。

### 改写已有目录

先保留编号和内容范围，标出结构问题，再改写标题。除非用户授权，不增加或删除主要知识单元。

### 评审目录

只诊断并提出修改建议，不直接替换全套目录。将问题区分为结构、内容边界、标题表达、篇幅和证据五类。

## 信息不足时的处理

- 缺少总字数：输出章节权重和建议区间，不输出看似精确的字数。
- 缺少案例或数据：使用方法型、问题型或场景型标题，并标记证据待补。
- 缺少作者口吻：默认采用通俗、克制、专业的市场书表达。
- 缺少小节范围：先从父级节的内容契约推导，再明确标注为建议结构。
- 章级框架存在明显重复：列出重复内容和影响，提供最小改动方案后继续。
- 选题边界含混：暂停生成标题，先确认读者、问题和结果。
