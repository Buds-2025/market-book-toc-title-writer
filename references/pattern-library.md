# 市场书爆款感标题公式库

## 使用方式

公式负责制造点击动机，语义卡负责限定内容边界。先从原题或正文中提取对象、痛点、动作、结果、场景、身份和专有名词，再隐藏原句并套入不同公式。禁止拿原题逐词替换。

每个内容单元至少生成六个候选，分别测试获得、摆脱、反差、降门槛、身份、损失等动机。最终标题只保留一个主钩子。

## 结果前置：先让读者看见所得

- `[结果]，现在可以从[一个低门槛动作]开始`
- `把[输入或旧状态]变成[可交付成果]`
- `[任务]做到这一步，[结果]才真正出现`
- `从[混乱状态]到[清晰结果]，[方法]重新安排了流程`

适合教程、方法、工作流和能力建设。结果必须具体到交付物、判断、选择或工作变化。

## 痛点释放：结束读者熟悉的低效

- `[高频任务]别再[低效动作]，[新方法]直接解决[具体问题]`
- `[场景]最耗人的一步，终于可以交给[工具或方法]`
- `还在为[痛点]返工？先改掉[具体环节]`
- `[问题]反复发生，通常卡在[被忽略的变量]`

适合办公、沟通、管理和交付场景。痛点必须是目标读者真实经历，不能制造焦虑。

## 反差纠偏：让旧认知失去稳定性

- `会[常见能力]只是起点，[系统能力]开始拉开差距`
- `[常见做法]越多，[隐藏代价]越难控制`
- `比[表面优势]更值钱的，是[真正决定结果的能力]`
- `[旧规则]正在失效，[新规则]把价值转向[新位置]`

适合趋势、观点、误区和决策。反差双方必须处于同一维度，正文需要解释变化机制。

## 降低门槛：把原来困难的结果交给普通人

- `不会[专业技能]，也能完成[具体成果]`
- `一句[输入]，直接做出[成果]`
- `原来要[多人或长流程]的工作，现在一个人也能[结果]`
- `不用从零开始，[工具或方法]先把[第一阶段成果]交出来`

适合AI工具、编程、设计、数据和视频章节。只能写工具实际支持的门槛变化。

## 身份跃迁：让读者看见新的角色位置

- `普通职场人也能拥有一支[专业团队或能力组合]`
- `一个人调度[资源组合]，照样完成[复杂交付]`
- `从执行者到[新角色]，[能力]改变了工作位置`
- `[职业身份]的新竞争力：能[高价值能力]，还能[结果责任]`

适合职场成长、自由职业、超级个体和一人公司。身份变化需要由可学习能力支撑。

## 损失与代价：让忽视成本变得可见

- `还在[旧做法]的人，正在多付出[具体代价]`
- `[环节]不先解决，后面的[结果]都会反复返工`
- `忽略[变量]，[任务]越快做完越容易出错`
- `[问题]拖到最后，最先失去的是[质量、时间或机会]`

适合风险、复盘、质量和边界内容。只描述可解释的现实代价，不放大正常试错。

## 资产沉淀：把一次劳动变成反复复用

- `把[个人经验]装进[模板、Skill或系统]，下一次直接调用`
- `[一次性产出]变成[长期资产]，工作开始越做越省`
- `别让[成果]用完就丢，把它沉淀成[可复用资产]`
- `从[零散材料]到[资产库]，[方法]让经验可以复制`

适合模板、知识库、Skill、工作流和组织资产。标题必须说明沉淀对象和复用途径。

## 流程交付：从单点功能走向完整结果

- `从[起点]到[终点]，[工具或方法]把整条流程跑通`
- `[步骤A]、[步骤B]、[步骤C]接起来，结果才真正交付`
- `一个项目装下[关键环节]，复杂任务不再四处分散`
- `让[多个角色或Agent]同时开工，[复杂成果]开始并行交付`

适合综合任务、项目管理、多Agent协作和案例实战。流程标题应与单点功能标题形成清楚区别。

## 强场景：让读者立即代入

- `[高压场景]来了，[工具或方法]先帮你守住[结果]`
- `[时间节点]不用再临时补，[日常动作]已经把材料备好`
- `面对[限制条件]，先完成[最重要的成果]`
- `[角色]遇到[高频难题]，这套方法直接处理[具体环节]`

适合通知、会议、述职、客户沟通、发布和交付检查。

## 品牌锚点：让产品名承担解决方案价值

- `[产品]真正改变的，是[工作方式或能力边界]`
- `用[产品]组建[新工作系统]，[目标读者]开始拥有[结果]`
- `[产品能力]接入[场景]后，[旧流程]被重新安排`
- `[产品]＋[关键能力]：把[复杂任务]交给一套系统`

品牌名只有在内容确实依赖该产品时使用。产品名后必须跟读者变化，不能只充当热词。

## 公式组合规则

1. 一个标题只使用一个主公式，第二分句只补充结果、证据或范围。
2. 同一章内不得连续三节使用相同开头、相同钩子或相同标点结构。
3. 至少一半标题应把结果、冲突、身份或代价放在前半句，避免全部从操作步骤开头。
4. “快速、轻松、清楚、更高效”不能单独承担钩子。
5. 数字、时限、案例、官方能力和效果必须有内容依据。

## 语义脱壳示例

原题：`多人多Agent协作，合理分工效率高`

语义卡：目标读者可调度多名Agent；不同Agent承担不同专业任务；结果是复杂任务可以协同推进。

可用候选：

- `一个人调度一支AI团队，复杂任务也能分工推进`
- `复杂工作别再单打独斗，多个Agent已经可以协同交付`
- `当Agent开始各司其职，普通人也有了自己的专业团队`

这些候选继承内容语义，没有继承原题的句法和行文节奏。

原题：`个人方法转化为可复制的数字能力`

语义卡：个人经验可以被结构化沉淀；载体可能是模板、规则、知识或Skill；目标是复用。

可用候选：

- `别让经验只留在脑子里，把它做成随时调用的Skill`
- `高手的方法开始可以复制：个人经验正在变成数字资产`
- `一次做对还不够，能把方法沉淀下来才会越做越快`
